# Task Decomposer

## Назначение
Разбивает сложные задачи на упорядоченные подзадачи перед выполнением.

## Когда использовать
- Задача от владельца требует более 3 шагов
- Автономная работа на маркетплейсе
- Архитектурные решения
- Задачи с зависимостями между шагами

## Алгоритм
1. **Анализ** — определить конечный результат и критерии успеха
2. **Декомпозиция** — разбить на атомарные подзадачи (1 инструмент = 1 подзадача)
3. **Зависимости** — определить порядок (какие задачи параллельны, какие последовательны)
4. **Оценка** — примерная стоимость в токенах каждой подзадачи
5. **План** — представить владельцу или выполнить автономно (если бюджет позволяет)

## Формат плана
```
Задача: [описание]
Подзадачи:
1. [подзадача] — инструменты: [список] — зависит от: [-]
2. [подзадача] — инструменты: [список] — зависит от: [1]
...
Оценка токенов: ~X
Время: ~Y мин
```

## Инструменты
- Plane MCP — создание issues для подзадач
- Sequential Thinking MCP — для сложной декомпозиции
- token_budget.py — проверка бюджета

## Правила безопасности
- Не выполнять необратимые действия без плана
- Бюджет > $1 — представить план владельцу
- Автономные задачи — максимум 10 подзадач
