# Secret Leak Scanner

## Назначение
Проверяет исходящие данные на наличие случайно попавших секретов. Работает на выходе (в дополнение к SecurityValidator на входе).

## Что искать

### API ключи и токены
- Паттерн: `sk-`, `pk-`, `api_`, `key_`, `token_`, `bearer `
- Длинные hex/base64 строки (>= 32 символа)
- JWT токены (`eyJ...`)

### Криптоключи
- Приватные ключи: `0x` + 64 hex символа
- Seed phrases: 12/24 слова из BIP39
- `-----BEGIN PRIVATE KEY-----`

### Пути и конфигурация
- `/app/payments/keys/`
- `.env` содержимое
- Пароли в открытом виде
- IP адреса сервера (если не публичные)

### Персональные данные владельца
- Telegram ID владельца (задаётся через env: OWNER_TELEGRAM_ID)
- Личный email
- Номер телефона

## Действия при обнаружении
1. **НЕ отправлять** сообщение/файл
2. Заменить секрет на `[REDACTED]`
3. Логировать попытку утечки (без самого секрета)
4. Если критично — уведомить владельца

## Исключения
- Публичный EOA адрес кошелька — можно (это публичная информация)
- Публичный BTC адрес — можно
- Имя сервера (hostname) — контекстуально
