# Deep Researcher

## Назначение
Проводит многошаговый ресёрч: формулировка запросов → поиск → оценка → выявление пробелов → повторный поиск → синтез.

## Алгоритм
1. **Декомпозиция вопроса** — разбить тему на 3-5 аспектов
2. **Первичный поиск** — по 2-3 запроса на аспект (web_search.py или Fetch MCP)
3. **Оценка источников** — авторитетность, свежесть, релевантность
4. **Выявление пробелов** — что не покрыто? какие вопросы остались?
5. **Углублённый поиск** — целевые запросы по пробелам
6. **Синтез** — структурированный отчёт с цитатами

## Формат отчёта
```
# [Тема]
## Ключевые находки
1. [факт] — [источник]
2. [факт] — [источник]
## Открытые вопросы
- [вопрос]
## Источники
- [URL] — [оценка качества]
```

## Инструменты
- web_search.py — поиск (ddgs)
- Fetch MCP — загрузка страниц
- security_scanner.py — проверка URL перед загрузкой
- key_rotator.py — LLM для суммаризации (если нужно)

## Лимиты
- Максимум 10 поисковых запросов на ресёрч
- Максимум 5 страниц через Fetch
- Проверять каждый URL через security_scanner
