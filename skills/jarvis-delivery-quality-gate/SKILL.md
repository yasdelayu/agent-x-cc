# Delivery Quality Gate

## Назначение
Финальная проверка качества работы перед сдачей на маркетплейсе. Предотвращает репутационный урон от некачественных deliverables.

## Чек-лист

### 1. Полнота (обязательно)
- [ ] Все требования из ТЗ покрыты
- [ ] Нет пропущенных пунктов
- [ ] Объём соответствует ожиданиям

### 2. Корректность (обязательно)
- [ ] Факты проверены (fact-checker для текстов)
- [ ] Код компилируется/запускается (для кода)
- [ ] Нет логических ошибок

### 3. Форматирование
- [ ] Структура чёткая (заголовки, списки)
- [ ] Нет опечаток в ключевых местах
- [ ] Соответствует запрошенному формату (markdown/json/код)

### 4. Безопасность
- [ ] Нет секретов в выходных данных (secret-leak-scanner)
- [ ] Нет личной информации Босса
- [ ] Код без уязвимостей (OWASP top 10 для code tasks)

### 5. Оригинальность
- [ ] Не скопировано дословно из источников
- [ ] Добавлена уникальная ценность

## Действия по результату
- Все ✅ → deliver
- Любой ❌ → исправить и перепроверить
- Невозможно исправить → уведомить Босса

## Инструменты
- fact-checker (skill) — для текстов
- secret-leak-scanner (skill) — для безопасности
- Bash — для тестирования кода
