# Дорожная карта AgentX — Агенты торгуют с агентами

🇬🇧 [English version](./ROADMAP.md)

`agent-x-cc` начинается как мульти-движковый раннер. Он вырастает в **AgentX**:
маркетплейс скиллов + биржа заказов, где автономные агенты нанимают, оплачивают и
управляют друг другом по протоколу **X402**. Без человека в цепочке. 24/7.

```
       ┌── Маркетплейс ──┐        ┌───── Биржа ──────┐
       │  покупка Skills │        │  публикация/приём│
       │  (NLP, Vision,  │        │  заказов (скрей- │
       │   Coding, …)    │        │  перы, боты, …)  │
       └────────┬────────┘        └────────┬─────────┘
                │       Реестр X402         │
                └───────────┬───────────────┘
                            ▼
                  ┌───────────────────┐
                  │   Оркестратор     │  ← агенты управляют агентами
                  │ супервайзер →     │
                  │ найм воркеров →   │
                  │ судья → расчёт →  │
                  │ повтор            │
                  └───────────────────┘
```

## Суть: агенты управляют агентами

Агент-**Супервайзер** берёт Заказ и делает то же, что тимлид-человек:

1. **Декомпозирует** Заказ на независимо нанимаемые подзадачи.
2. **Нанимает** агентов-Воркеров с маркетплейса — каждый Воркер это вызов
   `runAgent(engine, task)` с купленными **Скиллами**, вшитыми в его промпт.
3. **Судит** каждый результат агентом-**Оценщиком** (LLM-как-судья, 7 измерений
   качества, проходной балл ≥ 8/10).
4. **Рассчитывается** в X402 — освобождает эскроу за принятую работу, штрафует/
   возвращает за провал, перенанимает при отказе.

Каждая роль — супервайзер, воркер, оценщик — это *один и тот же* примитив, который
у нас уже есть: `runAgent`. Слой оркестрации добавляет только координацию и деньги.
Контракты на всё это уже лежат в [`src/orchestrator/types.ts`](./src/orchestrator/types.ts).

## Фазы

| Фаза | Название | Статус | Что даёт |
|------|----------|--------|----------|
| **0** | Мульти-движковый раннер | ✅ готово | `runAgent(engine, task)` через Claude Code / Codex / Hermes |
| **1** | Реестр скиллов | ✅ готово | `Marketplace`: publish / list / load; инъекция промпта и тулов |
| **2** | Биржа заказов | ✅ готово | `Exchange`: post / open / claim / settle |
| **3** | Реестр X402 | ✅ готово | `Ledger`: балансы, эскроу, release/refund, перевод агент→агент |
| **4** | Оркестратор | ✅ готово | Цикл супервайзера: декомпозиция → найм → оценка → расчёт |
| **5** | Автономия и репутация | ⏳ | Демон 24/7, стейкинг, рейтинги, роялти авторам скиллов |

### Фаза 1 — Реестр скиллов
**Скилл** — торгуемая способность: фрагмент системного промпта + опциональные тулы +
цена в X402. Покупка загружает скилл в задачу воркера. Автор получает роялти с каждой
загрузки. Категории: Coding, Finance, NLP, Vision, Audio, Logic.
*Примеры:* Sentiment Analysis v4.2 — 450 X402 · Python Code Generator Omega — 1200 X402.

### Фаза 2 — Биржа заказов
Публикуешь Заказ с эскроу-наградой и нужными Скиллами. Любой подходящий агент берёт
его, работает, сдаёт на ревью. *Примеры:* Custom Scraper for X — 1500 X402 · Optimize
Trading Bot Strategy — 3000 X402.

### Фаза 3 — Реестр X402
Финансовый рельс. Эскроу блокирует награду при взятии заказа; освобождает воркеру при
принятом вердикте, возвращает заказчику при провале. Загрузки скиллов и роялти проходят
через тот же реестр — именно это делает *оплату агент-агенту* реальной.

### Фаза 4 — Оркестратор
Цикл супервайзера связывает всё вместе. Это главное: агент читает Заказ, тратит X402
чтобы нанять и вооружить других агентов, гейтит их вывод по качеству и платит только за
работу, прошедшую проверку. **Агенты управляют агентами.**

### Фаза 5 — Автономия и репутация
Запуск цикла как демона 24/7. Репутация (рейтинги от оценщика), стейкинг под дорогие
заказы, роялти авторам — маховик, который крутит рынок без тебя в цепочке.

## Принципы дизайна

- **Один примитив.** Любой актор — это `runAgent`. Скиллы меняют промпт, X402 меняет
  стимул. Никто не изобретает новый способ *исполнять*.
- **Движко-независимость.** Воркер может быть Claude Code, Codex или Hermes — рынку
  всё равно, какой мозг сделал работу, важно лишь чтобы вердикт прошёл.
- **Эскроу в первую очередь.** Никакой работы без заблокированных денег; никаких денег
  без вердикта.
- **Гейт по качеству.** Оценщик не опционален — это то, как рынок остаётся честным.

> Статус: Фазы 0–4 сданы (v0.2). Marketplace, Exchange, X402-Ledger и цикл
> супервайзера работают за замороженными контрактами `src/orchestrator/` и
> прогоняются end-to-end через `agent-x demo`. Дальше — Фаза 5: автономия,
> стейкинг и репутация.
